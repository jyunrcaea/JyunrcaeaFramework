# Jyunrcaea! Framework에 대해

## 개발 계기
처음에는 'Jyunrcaea!' 라는 간단한 리듬게임을 만들려고 했습니다.   
파이썬으로 시작했으나... 성능문제와 코드가 더러워지면서,   
C#으로 갈아타게 되었습니다.   
   
유니티는 너무 싫고 쓰는법도 몰랐습니다.   
그래서 멀티미디어 라이브러리를 사용해서 만들려고 했죠.   
   
하지만 너무 수동적인 라이브러리인지라,   
규격이나 형식 같은게 없어서 코드가 자꾸 더러워지고 성능은 끔직해져갔습니다.   
심지어 코딩하기도 힘들었죠.
   
그러다 문뜩, osu! lazer를 업데이트 하러 깃허브 리포지토리를 뒤져보다가,   
osu! framework 라는걸 발견했습니다.   
그리고 나도 이런 프레임워크를 직접 만들어서 쓰면,   
유니티를 배울 필요도 없고 내가 원하는 기능을 추가할수도 있고   
완전 좋은 해결책이였습니다.   
   
그래서, 만들게 되었습니다...!   
Jyunrcaea! Framework
(느낌표는 필수입니다.)

## 역사
처음엔 SFML 을 이용해서 만들었습니다.   
하지만 SFML의 문제는 창을 조절할때 창이 렌더링 되지 않는다는 점이였습니다.   
   
제가 처음 osu! lazer를 실행했을때, 충격먹었던 점이 창을 조절하는 동시에도 계속 렌더링이 되는거였습니다.   
그래서 나도 창을 조절하면서도 렌더링이 되게끔 만들어야겠다는 생각밖에 안들었습니다.   
그후 SDL2을 많이 검색해서 알아보고 사용해보고 이것저것 경험을 많이 쌓았습니다.   
그리고 SDL2로 다시 재개발을 하게 되었습니다.   
전에 만들던 프레임워크에서의 문제점도 고칠겸...   
   
이전의 경험을 토대로 처음부터 어느정도 설계를 해둔뒤   
그걸 구축해가면서 현재의 안정적인 프레임워크가 만들어지게 되었습니다.   
심지어 사용하기도 쉽고, 개발환경 준비도 간단하죠!   
   
그리고 지금도, 완벽을 향해 개발하고 있습니다.

## 신념
Jyunrcaea! Framework의 신념은 총 3가지입니다.

- 자유를 먼저 추구한뒤, 가능한 그다음에 편의를 추구해라
- 사용자가 무엇을 하든, 제 기능을 다 해라.
- 의도하는 개발방식을 위해 막을껀 막되, 다른 길을 제공해라.

## 프레임워크 정보
실행시 .NET 6.0 Runtime 필요.
언어는 C# 10.0 으로 작성됨.
멀티미디어 라이브러리는 SDL2 를 이용함.
